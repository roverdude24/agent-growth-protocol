# Ultrawork Lane

This lane is the fast autonomous path for small, obvious tasks.

## When to use
- tiny fixes
- one-file edits
- short answers
- lightweight transforms
- narrow searches
- simple confirmations

## Operating style
- Minimize analysis.
- Pick the narrowest valid lane.
- Execute immediately.
- Verify once.
- Report the result with minimal evidence.

## Ultrawork rules
- If the task expands, promote to Prometheus.
- If the task needs multiple lanes, stop treating it as ultrawork.
- If the task needs repeated retries or broad investigation, it is no longer ultrawork.
- Keep the parent turn short and focused.

## Best tools
- terminal for direct execution
- search_files for quick lookup
- read_file for targeted inspection
- patch / write_file for tiny edits

## Good outputs
- exact result
- one verification step
- no extra ceremony
